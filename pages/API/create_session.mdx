# Create a Session

This section explains how to create a new session with the MultiOn Agent. A session in this context refers to a stateful instance of interaction with our Agent for a specific user query or workflow.

## Understanding Sessions

<br />
<center>
  {" "}
  <img src="../api.svg" alt="Add_to_chrome" style={{ width: "3000px" }} />{" "}

<br />

- **Session Lifecycle**: A session begins when you initiate it with a input url and prompt and ends whens the Agent has completed the task with the required Actions. Each session is isolated, ensuring that data and interactions are not shared across sessions.
- **Use Cases**: Sessions are ideal for handling individual queries, like fetching weather updates or executing a specific task.
- **Security**: Sessions maintain high levels of security, as each session is authenticated and isolated, protecting the integrity of your data and interactions.

## Session Input

- **input**: The user input query or command for the session. This should be a clear, concise statement of what you need.
  - Example: `"input": "Weather Today in SF"`
- **url**: The URL where the session will start from. This should be a valid, accessible URL.
  - Example: `"url": "https://www.google.com"`

### Optional Parameters

- You can also include additional parameters to refine your session's behavior and responses. Refer to the API documentation for a full list of optional parameters.

## Create Session

Create a new session by running the following code:

<Tabs items={['Python', 'JS', 'Curl']}>
  <Tab>
  ```python copy
  response = multion.new_session({"input": "Weather Today", "url": "https://www.google.com"})
  ```
  </Tab>
  <Tab>
  ```js copy 
  await multion.new_session({input: "Weather Today", url: "https://www.google.com"});
  ```
  <Callout type="info" emoji="ℹ️">
  Run the above code using **Node.js**
  </Callout>
  </Tab>
  <Tab>
  ```curl copy
  Pass either the <AUTH_TOKEN> or the <API_KEY> in the header.
curl --location 'https://api.multion.ai/sessions' \
 --header 'Content-Type: application/json' \
 --header 'Authorization: Bearer <AUTH_TOKEN>' \ # Use this for OAuth
--header 'X_MULTION_API_KEY: <API_KEY>' \ # Or use this for API Key
--data '{
"input": "Weather Today",
"url": "https://www.google.com"
}'
```
