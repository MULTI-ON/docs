Thank you for considering contributing to MultiOn! Your contributions are valuable to us and the community.

## Ways to Contribute

1. **Bug Reports:** If you encounter any bugs or issues, please [open an issue](../../issues) with detailed information.

2. **Feature Requests:** Have ideas for new features? Share them through [feature requests](../../issues).

3. **Pull Requests:** Contribute code by forking the repository, making changes, and submitting a pull request.

4. **Documentation:** Help improve documentation by suggesting edits or additions.

5. **Testing:** Test the project and provide valuable feedback.

## Getting Started

1. Fork the repository and clone it to your local machine.

2. Create a new branch for your changes:
```
    git checkout -b your-branch-name
```

3. Make your changes, commit them, and push to your fork:
```
git add .
git commit -m "Your commit message"
git push origin your-branch-name
```


4. Open a pull request (PR) against the `main` branch of the main repository.

## Code of Conduct

Please adhere to contribution guidelines and be respectful to all contributors and community members.
